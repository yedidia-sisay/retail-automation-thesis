# Odoo ERP — Retail Checkout Automation

This folder contains the Docker-based Odoo 17 deployment for the thesis demo.

Odoo acts as the official ERP/POS system.  
Django is the checkout coordinator.  
YOLO must never write directly to Odoo.

---

## Architecture rule

```
Camera → YOLO Service → Django Backend → Cashier UI
                                ↓  (confirmed orders only)
                           Odoo ERP/POS
```

Only cashier-approved, confirmed transactions are pushed to Odoo.  
YOLO detections are suggestions. The cashier decides what goes on the receipt.

## Quick start

### 1. Start the containers

```bash
cd erp
docker compose up -d
```

First run downloads ~1.5 GB of images and initialises the database.  
This takes 2–5 minutes. Watch progress with:

```bash
docker compose logs -f odoo
```

Wait until you see:
```
odoo  | INFO odoo_retail odoo.service.server: HTTP service (werkzeug) running on ...
```

### 2. Run the setup script

```bash
# From the erp/ directory
python scripts/setup_odoo.py
```

This script (no extra dependencies — uses stdlib `xmlrpc.client`):
- Creates the `odoo_retail` database if it doesn't exist
- Installs: `point_of_sale`, `stock`, `account`, `sale_management`
- Configures the company as "Retail Demo Supermarket" (Addis Ababa, ETB)
- Creates POS categories: Beverages, Packaged Products, Weighted Products
- Creates 21 products (17 YOLO classes + 4 weighted items) with barcodes
- Creates payment methods: Cash, Telebirr (Simulated), Chapa (Simulated)
- Creates POS terminal: "Cashier Terminal 1"
- Creates cashier user

### 3. Export Odoo product IDs

```bash
python scripts/export_odoo_ids.py
```

Writes `data/odoo_product_ids.json` — used by Django to populate  
`Product.odoo_product_id` fields.

### 4. Verify

```bash
python scripts/health_check.py
```

## Django integration

### How Django syncs products from Odoo

Django's `apps.erp.services.sync_products_from_erp()` calls  
`RealOdooClient.sync_products()` which fetches all POS-available products  
via XML-RPC and updates `Product.odoo_product_id` and `ERPProductMapping`.

To trigger a sync from Django:

```bash
# Via management command (add this if needed) or via the API:
POST /api/erp/sync-products/
```

Or use the exported JSON:

```bash
# In erp/
python scripts/export_odoo_ids.py
# Then in backend/
python manage.py shell -c "
from apps.erp.services import sync_products_from_erp
print(sync_products_from_erp(create_missing=False))
"
```

### How Django pushes confirmed receipts to Odoo

After a cashier confirms a transaction:

1. Django creates a `Receipt` with `payment_status=PAID`
2. The checkout view calls `push_receipt_to_erp(receipt_id=...)`
3. `RealOdooClient.push_receipt()` creates a `pos.order` in Odoo via XML-RPC
4. The Odoo order ID is stored in `Receipt.erp_reference`

To enable real Odoo integration, set in `backend/.env`:

```env
ERP_CLIENT_MODE=ODOO
ODOO_URL=http://localhost:8069
ODOO_DB=odoo_retail
ODOO_USERNAME=admin
ODOO_PASSWORD=admin
```

Leave `ERP_CLIENT_MODE=FAKE` for development/testing without Odoo running.

```bash
# Start
docker compose up -d

# Stop (data preserved)
docker compose down

# View logs
docker compose logs -f odoo

docker exec -it retail_odoo_db psql -U odoo -d odoo_retail
```

## Troubleshooting

**Odoo shows "Database not found"**  
→ Run `python scripts/setup_odoo.py` to create and initialise the database.

**Module installation hangs**  
→ First-time module install can take 3–5 minutes. Check `docker compose logs -f odoo`.

**XML-RPC authentication fails**  
→ Ensure the database exists and the admin password is `admin` (Odoo default).  
→ Check `ODOO_PASSWORD` in `backend/.env`.

**Port 8069 already in use**  
→ Change `ODOO_PORT` in `erp/.env` and restart.

**POS session won't open**  
→ Ensure at least one payment method is configured on the POS terminal.  
→ Go to Point of Sale → Configuration → Point of Sale → edit "Cashier Terminal 1".
